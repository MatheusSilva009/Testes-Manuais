**Testes Manuais — Repositório**

Bem-vindo! Este repositório reúne os cenários de testes manuais, evidências e anotações sobre bugs e falhas encontrados durante os testes.

**Resumo**:
- **Propósito:** Centralizar cenários de teste, evidências e problemas encontrados.
- **Escopo:** Testes manuais relacionados ao fluxo de login, inventário, carrinho e checkout.

**Estrutura do repositório**
- `Usuarios.feature` : Cenários relacionados a usuários.
- `features/` : Pasta com arquivos de cenário e `step_definitions`.
- `Evidências/` : Subpastas contendo capturas e relatórios por caso de teste (ex.: `CT 1 - Página inicial — usuarios e seus acessos`).
- `Readme.md` : Este arquivo (versão reestruturada).

**Como navegar**
- Abra a pasta `Evidências/` para ver capturas de tela e relatórios organizados por caso de teste.
- Revise `features/` e `step_definitions/` para entender os cenários descritos e possíveis passos reproduzíveis.

**Como contribuir / registrar um bug**
- Descreva claramente o passo a passo para reproduzir o problema.
- Inclua: ambiente (SO/navegador/versão), passos, resultado esperado, resultado observado e anexos (capturas em `Evidências/`).

**Boas práticas**
- Nomeie evidências com o código do caso de teste (ex.: `CT-03_login_fail.png`).
- Mantenha os cenários e definições de passo atualizados ao alterar fluxos.
